# Log Format

Use these formats for `.dev-session/` files.

## SESSION.md

Keep this short enough to read at the start of every session.

```markdown
# Dev Session

## Current State
- Branch:
- Goal:
- Last meaningful change:
- Dirty worktree:

## Next Start
1.
2.
3.

## Active Risks
- 

## Key Commands
- 

## Pointers
- Latest log:
- Latest snapshot:
- Important decisions:
```

## state.json

Structured index generated by `update_session.py` for quick recovery and script-friendly state:

```json
{
  "updated_at": "2026-05-28T12:00:00+02:00",
  "branch": "main",
  "dirty_worktree": "clean",
  "goal": "",
  "next_actions": [],
  "active_risks": [],
  "key_commands": [],
  "latest_log": ".dev-session/logs/YYYY-MM-DD.md",
  "latest_snapshot": ".dev-session/snapshots/YYYYMMDD-HHMMSS.md",
  "decisions": ".dev-session/decisions.md"
}
```

For folders that are not git repositories, `branch` and `dirty_worktree` may be `N/A (not a git repository)`.

## profile.json

Committed, non-secret workflow truth for the repo:

```json
{
  "provider": "github",
  "default_branch": "main",
  "ownership": {
    "workflow_owners": ["@org/dev-workflow-owners"],
    "lane_owners": {
      "frontend": ["@org/frontend-leads"],
      "backend": ["@org/backend-leads"],
      "mcp": ["@org/agent-platform"],
      "agentic": ["@org/agent-platform"],
      "iot": ["@org/iot-leads"],
      "ops-security": ["@org/platform-security"],
      "docs-product": ["@org/product-leads"]
    }
  },
  "mcp_trust_catalog": [
    {
      "name": "github",
      "capabilities": "read-write-provider",
      "required_env_vars": ["GITHUB_TOKEN"],
      "safe_smoke_test": "list current repository metadata",
      "owner": "@org/dev-workflow-owners",
      "risk_level": "medium"
    }
  ]
}
```

Never store tokens, raw credentials, private keys, or local-only auth state.

## Daily Log

Append chronological notes. Later starts on the same day may add a `Session resumed.` entry under `Work Log`.

```markdown
# YYYY-MM-DD

## Start
- Goal:
- Branch:
- Starting point:

## Work Log
- HH:MM - 

## Agent Runs
- 

## Verification
- 

## Handoff
- Changed:
- Why:
- Tests/checks:
- Decisions:
- Risks:
- Next action:
- Inspect first next session:
```

## Decision Entry

```markdown
## YYYY-MM-DD - Decision title

Decision:
Rationale:
Alternatives considered:
Impact:
Review trigger:
```

## Risk Entry

```markdown
## YYYY-MM-DD - Risk title

Risk:
Impact:
Mitigation:
Owner/next action:
Status:
```

## Agent Run Entry

```markdown
## YYYY-MM-DD HH:MM - Role / task

Agent:
Scope:
Outcome:
Files changed:
Commands run:
Follow-up:
```

## Flow Metric Entry

`flow-metrics.jsonl` is for workflow friction, not developer scoring:

```json
{"kind":"setup_gap","note":"missing GitHub MCP auth","privacy":"No developer names or performance scores.","recorded_at":"...","value":""}
```

Allowed kinds: `review_age`, `blocked_reason`, `flaky_check`, `setup_gap`, `handoff_gap`.

## Repository Policy Entry

Record whether `.dev-session/` is committed or local-only:

```markdown
## YYYY-MM-DD - Dev session repository policy

Decision:
Committed files:
Ignored files:
Secret/sensitive-data handling:
Rationale:
```
